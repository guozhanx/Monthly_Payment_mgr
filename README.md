# Monthly_Payment_mgr
架构介绍：
1.测试用例文件夹中的文件，记录了用例设计的思路和具体输入数据

2.Behavior中存放的是脚本行为

3.common存放通用公共方法，包括日志、查找最新报告、邮件发送测试报告等方法

4.Data存放测试数据，可能包括数据生成的函数或者csv、excel文件

5.images存放测试过程中的截图

6.report存放每次执行自动化测试用例生成的测试报告

7.TestCase存放测试用例脚本

8.WebApi存放页面的元素


关于执行用例：
登录->用例执行->退出
用例执行过程中不做过多的反向用例验证，如果有问题会抛出异常
